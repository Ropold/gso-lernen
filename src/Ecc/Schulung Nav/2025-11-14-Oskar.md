# Variablen

`<simple-var>.INIT`: Setzt Variablenwert auf Initialwert zurück

`<record>.RESET`: Setzt Ergebnissatz, Filter u.ä. zurück

# Functions

Im _View → C/AL Globals_ unter Functions anlegen

# Sprachkonstrukte

`IF <expr> THEN <stmt>`:

`IF <expr> THEN <stmt> ELSE <stmt>`:

`EXIT[(<expr>)]`: Direkter Abbruch der aktuellen Funktion

**Notiz:** Ein `EXIT` in einem Trigger unterbricht die Default Action.

`CASE <expr> OF <const> : <stmt> {; <const> : <stmt>} [ELSE <stmt>] END`

`BEGIN <stmt> {; <stmt>} END`: Anweisungsblock

`FOR <expr> (TO|DOWNTO) <expr> DO <stmt>`: In 1er-Zählschritten,
bis einschließlich Endwert

`WHILE <expr> DO <stmt>`: While-Schleife

`REPEAT <stmt> UNTIL <expr>`: "Until true" statt "while true";
läuft solange "until false"

**Notiz:** Zählschritt muss explizit angegeben werden.

# Namensgebung

Lokale: Präfix `L_`

Globale: Präfix `G_`

Record: UpperCamelCase des zugehörigen Tabellennamen

Hungarian Notation: Bedingt erlaubt, falls Datentyp
zur Anwendung tatsächlich relevant

# Funktionssignatur

Parameterliste einstellen: Funktion auswählen,
dann über _View → C/AL Locals_ in "Parameters"

**Notiz:** Bei Aktivierung von `VAR` sind Übergabeparameter
als Aliase (oder Referenzen) übergeben.

Return value: Unüblich aber möglich; eher über VAR zurückgeben

# Textkonstanten

Für i18n, unter Drilldown mehrsprachige Alternativen angeben.

**Notiz:** Diese immer global anlegen, nicht lokal.

# Sonstige Funktionen

`ERROR`: Selten verwendet, da Standard-Error bereits hilfreich sind

`CONFIRM`: Eine OkCancel MsgBox

`STRMENU(<text>,<def-opt>)`: Gibt eine Auswahl als RadioGroup

**Notiz:** Der Optionen sind im Text als CSV. Zeilenumbrüche
über `+/` bzw `/+` angeben (ausprobieren!).

`<record>.SETCURRENTKEY`: Einstellen der Sortierung der Tabelleneinträge

`<record>.SETRANGE(<field-name>[,<from>][,<to>])`:
Filtert die Tabelleneinträge

`<record>.SETFILTER(<field-name>,<text>)`: Genaue Filterangabe

`<record>.FIND`:

**Notiz:** Bevorzuge `findfirst` statt `find('-')`.

`<record>.GET(<expr>)`: Bekomme _einen_ Eintrag abh. vom KEY

**Notiz:** Gibt Fehlermeldung zurück bei Nicht-Fund.
Kann mit IF geprüft werden.

`<record>.(INSERT|MODIFY|DELETE)[(<do-validate>)]`: Einträge ändern;
standardmäßig die Validierung überspringen

`<record>.CALCFIELDS(<field-name>[,<field-name>])`:
Berechnet FlowFields für den Ergebnissatz

# Kommentare

`//...`: Einzeilig
`{...}`: Auch mehrzeilig
